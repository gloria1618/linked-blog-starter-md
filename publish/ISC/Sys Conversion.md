System Conversion Methods

- orgs have diff options when converting computer sys, like soft/hardware n data, from one info sys to another
- choice varies depending on unique needs
	- smaller org more likely to select more aggressive approach such as direct changeover
	- larger org more likely take more cautious path, parallel/phased approach
- Direct
	- org ceasing use of old, starting new immediately 
	- risk 
		- if new sys doesn't work, op severely hindered/stops depending on what sys supporting
- Parallel
	- new sys implemented while old sys till in use for extended time period
	- downside
		- req sig effort from personnel bc must op 2sys simultaneously 
- Pilot
	- performs conversion on small scall w test env, while cont use old 
	- allows validation/testing before roll out to entire org so can make adj prior to full implementation
- Phased
	- aka gradual/modular
	- gradually ads vol to new sys while still op old sys
	- useful for org w dist loc, allows to implement one site at a time
- Hybrid
	- custom combo, tailored to unique needs
