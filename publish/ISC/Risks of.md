
- Cloud Computing
	- most cloud providers req to maintain high sec due to regulations like GDPR, HIPAA
	- common for 3rd party cloud providers to have SOC 2 engagements, which are independent audits of
		- mgmts attestation regarding cloud service provider's ctrls
		- other mgmt claims abt sec over customer's data, privacy, confidentiality
		- other info may incl how ctrls address Cloud Sec Alliance's Cloud Ctrls Matrix
			- prov sec principles to guide cloud vendors
			- assists prospective cloud customers in assessing overall sec risk of cloud provider
			- utilize industry accepted sec standards, re, ctrls framework like COBIT, NIST, etc
	1. Additional Industry Exposure
		- exposed to unique industry risks of other shared cloud subscribing orgs
	2. Cloud Malware Injection Attacks
		- specific to cloud computing based sys
		- gain access to cloud env, inject malware
	3. Compliance Violations
		- providers dont have sec protocols/procedures to meet regulations
	4. Data Loss
		- susceptible, but less likely, to data breach/loss/expose
	5. Control Loss
		- no phys/logical access to computing equip > relinquish ctrl over infra
		- chngs/upgrades to cybersec measures may not be preferred timely/standard
	6. Visibility Loss
		- of full IT infra > ctrl loss
		- cloud provider has full visibility, subs org doesnt know all risks
	7. Multi Cloud/Hybrid Mgmt Issues
		- org subs to mult cloud based sol and/or maintains some on premise infra
		- can be good cybersec diversification
		- can be challenging to integrate/monitor mult env, making detecting cyberattack difficult
	8. Theft/Loss of Intellectual Property
		- risk provider lacks sufficient ctrls over data

- Mobile Tech
	- risks for users, emplee
	- mobility, lack of oversight > cybersecurity risks
		- exposed to public networks
			- install software to monitor/protect users from potential attacks
			- if protective software not used, policies prov guidelines
	- most mobile devices sim to computers in terms of func, type of info access
	- yet mobile device op sys diff from computer
		- separate patch/updates 
	1. Application Malware
		- user downloads app seeming legit but gives unauth user acces to device
	2. Lack of Updates
		- uninstalled patches/sec fixes yet to be installed leaving device vuln
	3. Lack of Encryption
		- many not encrypted, only rely on passcode
		- once access gained, passwords can be reset on web using victims email on mobile device 
	4. Physical Threats
		- loss/theft of mobile device
		- esp if device doesnt have sufficient access ctrls
	5. Unsecured WiFi Networks
		- public unsecured networks, anyone on same network could potentially access device, steal info, infect w malware 
	6. Location Tracking
		- unauth tracking using GPS to locate ppl, devices, As
		- devise plans using loc to do attack
	7. Other
		- mobile storage devices to infect like USB, mem card, optical disks
		- can be monitored/surveilled for eavesdropping
		- equip like cell-site simulators, Bluetooth scanners, IMSI international mobile subscriber identity catchers can be used to intercept call/txt w.in a radius


- IoT
	1. Device Mismgmt
		- insuff passwords, mismgmt inc attack risk > loss of info/access
	2. Device Spoofing
		- connect illegitimate device to network
	3. Escalated Cyberattacks
		- IoT devices used as attack base to infect more machines, or as entry pt to network
	4. Expanded Footprint
		- IoT paired w other devices directly connected to core network inc # of pts subject to attack 
	5. Info Theft
		- connected to internet > bc data stored in cloud or on other devices that can be accessed 
	6. Outdated Firmware
		- intercept IoT firmware updates, manipulate firmware w known weaknesses
		- esp since bring into office w.o alerting IT personnel, often missed as part of orgs cybersec defenses
	7. Malware
		- susceptible to cyberattacks bc limited computing power
	8. Network Attacks
		- launch DoS denial of service on IoT network/devices like trad networks
