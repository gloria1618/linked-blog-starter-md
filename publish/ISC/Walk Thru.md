Walk Thru of Org's Security, Confidentiality, Priv Procedures

- effective sec, confidentiality, priv pol/prac req comprehensive strat throut org
- to inc prob that strat executed, perform periodic read/walk thrus led by IT dep/joint task force w both IT/non
- esp in case of data breaches (confidentiality, priv), sec incident/disaster (security)

- Read thru
	- dist sec, conf, priv proc to mem of IT/non deps supporting walk thrus for rev
	- lets org
		- inform personnel of tactical/strat procedures
		- rev plans for obsolete info
- Walk thru
	- role playing/simulating disaster scenario where attacker attempting to obtain data
	- have team mem use proc to prevent attack, ensure data leaked cant be deciphered
	- more common than fire drills
	- involv team mem obtaining evidence abt whether documented policy req aligns w observed proc
	- phases
		1. Plan/Prep
			- def scope
			- id key ctrl/processes
			- id personnel
		2. Obtain Understanding
			- of process being eval
			- review documentation 
			- interview personnel
			- create notes
		3. Perform Walk Thru
			- reperform process
			- verify results/effectiveness
		4. Create Documentation 
			- create workpapers
			- document proc
		5. Test
			- test ctrls id in walk thru
			- obtain samples if needed
		6. Eval/Report
			- eval proc, report on effectiveness
			- interpret results
			- prep report to summarize findings
			- prov recc
- Fire drill
	- less common, but effective way to test sec pol and det if emplee
		- understands emergency plans
		- follow proper protocol when believe an emergency situation is occurring
